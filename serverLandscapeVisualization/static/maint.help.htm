Maintenance Page Handler - Help


Pre-condition on the server:
The two files
/app/www/apache2/htdocs/maintenance/site-maintenance-page/maintenance.html
/app/apache2-8080/htdocs/maintenance/site-maintenance-page/maintenance.html
and their folder-structure should be writable by the linux user running the Apache WebServer.
In case of errors, an error message appears, like

In case such error is received, please ensure that the access rights on the specified folder & the specified file must be corrected.
The current Production cluster is marked on the page with the text "PROD ENVIRONMENT":
Additional details about Cluster A:
The maintenance page on CLUSTER A will be activated, only if the file
/app/www/apache2/htdocs/maintenance/site-maintenance-page/maintenance.html
exists on the server mchp486a.

The activate/deactivate button of CLUSTER A renames the above file (places or removes '_' at the beginning of the file name)
Additional details about Cluster B:
The maintenance page on CLUSTER B will be activated, only if the file
/app/apache2-8080/htdocs/maintenance/site-maintenance-page/maintenance.html
exists on the server mchp486a.

The activate/deactivate button of CLUSTER B renames the above file (places or removes '_' at the beginning of the file name)
Manual activation/deactivation of the Maintenance Pages:
You can rename manually the above files on the linux command line on the server mchp486a to activate / deactivate the maintenance page(s)
(if your user rights are sufficient for this operation):
- If you place the _ character on the beginning of the file name, the maintenance page will be deactivated.
- If you remove the _ character from the beginning of the file name, the maintenance page will be activated.
